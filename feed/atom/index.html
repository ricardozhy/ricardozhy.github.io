<p>ITIMER_PROF：进程在用户模式（即程序执行时）和核心模式（即进程调度用时）均计数。定时到达产生SIGPROF信号。ITIMER_PROF记录的时间比ITIMER_VIRTUAL多了进程调度所花的时间。</p> <p>定时器在初始化是，被赋予一个初始值，随时间递减，递减至0后发出信号，同时恢复初始值。在任务中，我们可以一种或者全部三种定时器，但同一时刻同一类型的定时器只能使用一个。</p> <p>用到的函数有：</p> <p>#include \&lt;sys/time.h&gt;<br> int getitimer(int which, struct itimerval *value);<br> int <p>chrome ://flags//# cookies-without-same-site-must-be-secure</p> <p>将这两项都设置成disable就可以啦</p> <p>2）Chrome94以上设置了samesite属性默认打开，开发者无法手动设置none，所以Chrome94以上的浏览器是用不了了</p> <p>3）Chrome80以下的应该是无需设置</p> <p>4）使用火狐浏览器可以直接打开，无需设置</p> <p>比较chrome和firefox的请求标头和响应标头：</p> <p>firefox :在首次启动请求后，服务端返回会话id，然后为请求中的每个cookie带来会话id。</p> <p>chrome :由于请求标头始终没有sessionId，且整个cookie为空，因此服务每次都无法接受sessionId，每次都重新分配session。</p> <p><strong>还有一些关于cookie，session，token之间有什么关系和区别的知识</strong></p> <p>1）cookie数据存放在客户的浏览器上，session数据放在服务器上。</p> <p>2）cookie不是很安全，别人可以分析存放在本地的cookie并进行cookie欺骗，考虑到安全应当使用session。</p> <p>3）session会在一定时间内保存在服务器上，存储在服务器的数据会更加的安全，不容易被窃取。当访问增多，会比较占用你服务器的性能与资源，应当使用cookie。</p> <p>4）单个cookie保存的数据不能超过4K，很多浏览器都限制一个站点最多保存20个cookie。因此使用cookie只能存储一些小量的数据。</p> <p>5）session和session_id：服务器会保存一份，可能保存到缓存/数据库/文件，session_id一般是随机字符串，要到服务器检索id的有效性。Session通过cookie存储session_id，然后具体的数据则是保存在session中。如果用户已经登录，则服务器会在cookie中保存一个session_id，下次再次请求的时候，会把该session_id携带上来，服务器根据session_id在session库中获取用户的session数据。</p> <p>6）token：服务器不需要记录任何东西，每次都是一个无状态的请求，每次都是通过解密来验证是否合法。</p> <h2>5.快速使用</h2> <p>1）使用火狐浏览器或谷歌80以下打开<a href="https://console.cloud.tencent.com/">腾讯云·控制台</a>, 点击右上角工具，cloudshell。</p> <p>2）打开<a
